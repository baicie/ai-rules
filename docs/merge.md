# Merge Strategies

airules writes generated content through merge strategies.

## managed-block

Use for shared markdown targets such as `AGENTS.md`.

```md
<!-- airules:start pack="@baicie/react-shadcn" install="codex" version="0.1.0" hash="..." -->

...

<!-- airules:end pack="@baicie/react-shadcn" install="codex" -->
```

Only content inside the managed block is updated.

## overwrite-managed

Use for generated files such as `.cursor/rules/*.mdc`.

The file is overwritten only when it is already airules-managed or was generated by airules.

## skip-if-exists

If the target exists, airules skips it.

Skipped files are not recorded in lockfile.

## manual

Generated output is written to:

```txt
.agents/agent/staged/
```

Users merge it manually.

## Conflicts

When airules detects user edits, it must preserve user content and avoid destructive writes unless the user explicitly forces the operation.
